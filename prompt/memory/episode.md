# 未完成任务执行历史与类似任务检索提示词
## 一、当前场景与核心任务
以上是需要推进完成的任务情况，需检索两类信息支撑执行：1. 本任务已产生的执行历史；2. 同类型/同领域任务经验，规避试错、优化路径。


## 二、检索核心目标
1. 提取本任务实时历史：明确进度、已解决/待解决问题，确保后续衔接无遗漏；
2. 定位类似任务经验：筛选可复用方法、风险策略，助力突破卡点。


## 三、重点检索内容（含支撑价值）
### （一）当前未完成任务执行历史
1. **执行阶段与已推进步骤**
   - 检索：当前阶段、已完成关键步骤及时间/交付物；
   - 价值：避免重复执行，防止遗漏衔接环节。

2. **当前卡点与尝试动作**
   - 检索：具体卡点、已尝试的解决动作及结果；
   - 价值：精准定位需突破问题，匹配类似任务解决方案。

3. **联动角色与资源**
   - 检索：已参与角色、投入资源及联动进度；
   - 价值：避免重复联动，防止资源冲突。


### （二）其他类似任务经验
1. **类似任务筛选标准**
   - 检索：同类型/同目标/同领域的任务特征；
   - 价值：确保经验关联性，排除无关领域信息。

2. **执行路径与关键动作**
   - 检索：核心推进路径、关键动作优先级；
   - 价值：复用优化执行逻辑，提升效率。

3. **问题解决经验**
   - 检索：同类卡点的解决方案及效果；
   - 价值：减少探索时间，快速突破当前卡点。

4. **资源与风险预警**
   - 检索：资源投入量、历史风险及应对措施；
   - 价值：合理规划资源，提前规避已知风险。


## 四、检索优先级
1. 第一：本任务“执行阶段+已推进步骤+当前卡点”；
2. 第二：类似任务“同类卡点解决方案”；
3. 第三：本任务“联动角色/资源”、类似任务“执行路径/风险措施”。
