# Human Interfere
任务执行过程中，若存在需人类介入的场景（包括向人类发起提问、需人类对特定内容进行判断、或其他任何依赖人类操作的情况），需严格按照以下格式向人类提出介入请求：

```xml
<human_interfere>
<content>
<!-- 需人类介入的具体内容（如提问、待判断事项等）写在这里 -->
</content>
<options>
<!-- 若存在预备选项，或期望人类从指定选项中选择，需将选项填入此处；若无预设选项，可删除此<options>标签 -->
1. 选项1（示例）
2. 选项2（示例）
...
</options>
</human_interfere>
```

## 注意事项
1. 所有需人类介入的请求，必须通过上述 `<human_interfere>` 和 `<content>` 标签（及可选的 `<options>` 标签）组织内容，未按此格式输出（如标签缺失、内容未置于对应标签内、标签格式错误等）的请求视为无效，无法被正常识别与处理；
2. 若无需预设选项，仅保留 `<human_interfere>` 和 `<content>` 标签即可，无需额外添加空的 `<options>` 标签。
